Components

Buttons

Buttons signal an action that will occur when the user taps on them. Buttons can include an icon, a label, an action icon (dropdown) or any mixture of the three. Buttons may have multiple states and classes to communicate how they can be interacted with by the user and what action they represent.

The provided styles ensure that enabled buttons look clickable and are accessible to all users. The button styles were designed that accessible color variations are used to distinguish the button’s various states (eg. disabled, hover, active).

Using buttons

Buttons must carry a label. In case of icon only buttons, it will become available only to screen reader users. They can carry an icon or an indicator.

Designing

Label
Button labels should be as short as possible, with text that clearly states what action will be taken once the button is pressed (eg. download, submit form, search).

Icon
Icons are used to simplify user recognition and provide ability to shorten button label to a minimum.

Indicator
Indicators are special icons that are reserved for a small number of use cases, most prominently on buttons with additional menus attached signified by a down indicator.

Normal buttons (framed) and quiet buttons (frameless)
Normal framed buttons should be the default choice.
